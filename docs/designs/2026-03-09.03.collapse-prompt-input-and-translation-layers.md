# Collapse Prompt Input And Translation Layers

## Design Goal

Reduce prompt-path complexity by removing translation-only layers and narrowing the prompt subsystem boundary. The prompt compiler should consume only model-visible data, not raw runtime/session transport shapes.

## Agreed Architecture

The prompt subsystem will use three main types:

- `PromptInput`
  - the only input to prompt compilation
  - contains only model-visible data
- `PromptEvent`
  - the normalized event type used by prompt compilation
  - replaces direct prompt-layer handling of raw `HistoryEvent`, `Trigger`, and payload-lookup side channels
- `CompiledPrompt`
  - provider-ready messages plus prompt diagnostics

The prompt subsystem entrypoint will become a real `PromptCompiler` object rather than a pass-through assembler wrapper.

## Boundaries

### Runtime / Session Layer

Owns:

- `SessionState`
- raw `pb::Trigger`
- raw `HistoryEvent`
- execution state
- payload lookup state
- compaction state

Does not expose raw runtime records directly to the prompt compiler.

### Prompt Preparation Layer

Owns:

- building `PromptInput`
- normalizing runtime/session records into `PromptEvent`

This layer is responsible for translating:

- committed history -> transcript prompt events
- queued trigger state -> pending prompt events
- resolved payload lookups -> payload lookup prompt events
- retry feedback -> retry prompt events

### Prompt Compiler Layer

Owns:

- stable prefix rendering
- prompt event transcript rendering
- timeline compaction
- message chunking
- prompt diagnostics

It should not know about:

- raw `pb::Trigger`
- raw `HistoryEvent`
- session internals not visible to the model

## Session / Prompt Model

### PromptInput

`PromptInput` should contain:

- `stable_prefix`
- `transcript_events`
- `pending_events`
- `compaction_blocks`
- optional `retry_feedback`

It should not contain runtime-only fields.

### PromptEvent

The normalized prompt event surface is:

- `UserMessage`
- `AssistantOutput`
- `ExecutionRequested`
- `AwaitedExecutionSucceeded`
- `AwaitedExecutionFailed`
- `ExecutionDetached`
- `DetachedExecutionSucceeded`
- `DetachedExecutionFailed`
- `ExecutionRejected`
- `PayloadLookupAvailable`
- `RetryFeedback`

`ContractDelta` is intentionally excluded for now.

### Stable Prefix

The stable prefix remains:

- `Harness Contract`
- `Identity Envelope`
- `Session Baseline`

The prompt compiler should render these from a smaller prompt-visible input contract, not from a broader runtime snapshot.

## Layer Changes

### Remove

- `PromptAssembler` as a pass-through wrapper
- prompt dependence on full `TurnSnapshot`
- prompt-layer translation from raw triggers and raw history records

### Keep

- `PromptMessage` as the provider-neutral model-input boundary
- `ModelDeltaEvent` as the provider-neutral model-output boundary
- client `EventRecord` as a real client semantic layer
- shared `fathom-protocol` wire contract

### Collapse / Replace

- replace prompt use of `TurnSnapshot` with `PromptInput`
- replace mixed prompt-side raw inputs with normalized `PromptEvent`
- replace the giant free-function prompt pipeline with a real `PromptCompiler`

## Prompt Compiler Shape

Recommended internal structure:

- `PromptCompiler`
  - orchestrates compilation
- `StablePrefixRenderer`
  - renders harness contract, identity envelope, session baseline
- `TranscriptBuilder`
  - works only from normalized `PromptEvent`
- `TimelineCompactor`
  - applies token-budget compaction
- `MessageChunker`
  - produces provider-ready messages
- `PromptDiagnosticsBuilder`
  - computes stats, hashes, and debug metadata

This should use plain Rust structs and methods, not a heavy trait-object hierarchy.

## Key Invariants

- The prompt subsystem must consume only prompt-visible data.
- Raw transport/runtime shapes must remain below the prompt boundary.
- Every prompt-visible event after normalization must be additive and immutable.
- The prompt compiler must not depend on runtime-only convenience fields.
- Layers are allowed only when they own a real semantic boundary.
- Translation-only layers should be removed.

## Rejected Alternatives

- Keep `TurnSnapshot` as the prompt input
  - rejected because it is too broad, too runtime-shaped, and already carries unused fields
- Keep `PromptAssembler` as-is
  - rejected because it is only a pass-through wrapper and hides the real complexity in `prompt.rs`
- Let the prompt compiler keep translating raw triggers/history directly
  - rejected because it couples prompt rendering to runtime transport/storage details
- Introduce `ContractDelta` into `PromptEvent` now
  - rejected as premature for the current cleanup target

## Open Issues

- Whether `TurnSnapshot` should be deleted entirely or retained for another runtime use after the prompt cutover
- Exact module layout for the new prompt compiler subsystem
- Whether prompt diagnostics remain attached to `CompiledPrompt` or move to a separate diagnostics object

## Next Implementation Slice

1. Introduce `PromptInput`, `PromptEvent`, and `CompiledPrompt`.
2. Replace prompt compilation input from `TurnSnapshot` to `PromptInput`.
3. Remove `PromptAssembler` and introduce a real `PromptCompiler`.
4. Move raw trigger/history/payload-lookup normalization out of `prompt.rs` and into prompt preparation.
5. Delete unused prompt-only residue such as `in_flight_actions` if no remaining prompt path needs it.
