## Context
This slice derives from docs/designs/2026-03-09.00.agent-prompt-context-and-execution-model.md. The current runtime and prompt inputs still reflect the older mixed prompt model, where durable session material, time context, triggers, and other mutable state are bundled together in TurnSnapshot and prompt assembly inputs.

The new design requires the prompt-facing input model to be re-centered around a stable prefix consisting of Harness Contract, Identity Envelope, and Session Baseline, with append-only transcript and execution events carried separately.

## Why It Is Needed
This is the foundation for the entire prompt/context rework. Until the runtime exposes the new stable-prefix layers explicitly, prompt assembly cannot be rewritten cleanly, execution events cannot attach to the new event log model, and compaction cannot rebuild the prompt from authoritative state. Evidence lives in the finalized design note and in the current mixed input path in fathom-server/src/runtime/context_snapshot.rs and fathom-server/src/agent/types.rs.
