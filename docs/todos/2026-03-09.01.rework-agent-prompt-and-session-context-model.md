## Context
This is a separate workstream from the earlier harness refactor. The current Fathom prompt bundle still rebuilds mixed stable and volatile context together on every turn in fathom-server/src/agent/prompt.rs and fathom-server/src/runtime/context_snapshot.rs. Stable identity and capability material is mixed with mutable state such as current time, in-flight actions, retry feedback, scheduler triggers, and resolved payload lookups.

The user wants to rework the model-visible session shape so the agent acts as the session decision maker for the live runtime environment, not merely a responder over engaged-environment metadata. The target direction is a cache-friendly prompt with a stable prefix and append-only turn/transcript items.

A comparison against the public open-source Codex repo shows the relevant pattern: stable instructions are carried separately from typed transcript input items, and turn-to-turn context changes are emitted as incremental updates rather than being folded back into a rebuilt static prefix. Fathom should adopt that separation while being stricter about keeping highly volatile values out of the reusable prefix.

## Why It Is Needed
This rework is needed because the current prompt composition is not cache-friendly and does not cleanly express the agent's real job. Today the prompt builder in fathom-server/src/agent/prompt.rs includes volatile fields like time context and resolved lookup payloads in normal prompt assembly, which makes stable-prefix reuse weaker and tangles long-lived identity/session semantics with per-turn runtime state.

A dedicated rework will let Fathom separate stable instructions from live context deltas, express user/assistant/tool exchange as a structured transcript, and make future prompt iterations cheaper and safer. Public Codex patterns support this direction: stable instructions stay constant while per-turn environment and policy changes are appended as input items, and request reuse depends on the non-input request shape remaining stable.

## Completion Criteria
- [ ] Choose and document the new agent-facing role framing and naming for the decision-making prompt persona.
- [ ] Split prompt/context composition into a stable prefix and turn-scoped volatile update items.
- [ ] Remove mutable values such as current time, retry feedback, in-flight actions, triggers, and resolved payload lookups from the stable prefix path.
- [ ] Replace the current synthetic timeline-oriented prompt sections with a structured transcript model that can represent user messages, assistant messages, tool calls, tool results, and system/session updates.
- [ ] Define the boundary between stable session contract data and live environment/runtime state, and reflect it in TurnSnapshot or equivalent prompt inputs.
- [ ] Add tests that verify stable-prefix consistency, turn-delta emission, transcript ordering, and prompt-shape behavior across consecutive turns.
- [ ] Update architecture or prompt-design documentation to describe the new model-visible context shape.
