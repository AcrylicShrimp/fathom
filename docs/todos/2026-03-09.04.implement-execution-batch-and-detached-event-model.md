## Context
This slice derives from the finalized execution design in docs/designs/2026-03-09.00.agent-prompt-context-and-execution-model.md. The runtime currently exposes older tool lifecycle and task-flow behavior that does not match the agreed batch-settle semantics, `await` / `detach` mode handling, or the finalized six execution event types.

The implementation needs to align capability metadata, tool dispatch behavior, agent wakeup policy, and model-visible events with the agreed execution model.

## Why It Is Needed
Execution semantics are central to the new harness contract. The agent must be able to choose legal tool modes, rely on one wakeup after batch settle, and reason over a small, stable execution-event surface. Without this work, the new prompt model and the runtime behavior would contradict each other.
